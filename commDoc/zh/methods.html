方法接口

设计器操作接口

这些接口仅用来对设计器实例进行操作体验调整、模拟手工操作工作区里的流程图。

this.setNodeRemarks() 返回值：void

当初始化后存在左侧工具栏时，设定左侧工具栏中每一种节点或按钮的说明文字。(仅Js原生版使用)

传参：( remarks )

名称类型/格式描述
remarksJsonMap key为按钮类型名，value为用户自定义文字说明，详见GooFlow.remarks.toolBtn。

this.setHeadToolsRemarks() 返回值：void

当初始化后存在顶部栏时，设定顶部栏中每个按钮的说明文字。(仅Js原生版使用)

传参：( remarks )

名称类型/格式描述
remarksJsonMap key为按钮类型名，value为用户自定义文字说明，详见GooFlow.remarks.headBtn。

this.setExtWorkRemarks() 返回值：void

设定扩展工作区宽高的长条按钮的说明文字。(仅Js原生版使用)

传参：( remarks )

名称类型/格式描述
remarksJsonMap 有两个String型属性：extendRight的值为工作区向右扩展的说明文字，属性extendBottom的值为工作区向下扩展的说明文字。

this.switchToolBtn() 返回值：void

当初始化后存在左侧工具栏时，切换选择工具栏中指定的按钮。

传参：( type )

名称类型/格式描述
typeString表示切换成哪种类型的按钮。

this.setDragAddOper() 返回值：void

是否设置当前添加块元素的操作方式为拖拽式。
要使该方法有效，需扩展引入dragadd.js文件。

传参：( bool )

名称类型/格式描述
boolBoolean为true时使用类似Visio的拖拽式；为false时使用类似Photoshop的选定类型后的点击式，默认为false。

this.focusItem() 返回值：void

选定某个节点/连线。

传参：( id, bool )

名称类型/格式描述
idString要选中元素的id。
boolBoolean是否要触发选中事件钩子函数，多用在程序内部调用。

this.blurItem() 返回值：void

取消所有节点/连线被选定的状态。

传参：(无)

this.copyNode() 返回值：void

复制一个节点详细信息以供接下来的粘贴操作。

传参：( id )

名称类型/格式描述
idString要复制节点的id。

this.parseNode() 返回值：void

粘贴一个最近被复制的节点。

传参：(无)

this.reinitSize() 返回值：void

重构整个流程图设计器的宽高，在浏览器窗口大小改变或者父容器宽高改变时，执行这个方法能让设计器重新适应新的宽高显示。

传参：( width, height )

名称类型/格式描述
widthNumber设计器新的宽度，单位像素。
heightNumber设计器新的高度，单位像素。

this.resetScale() 返回值：void

重设整个工作区内容的显示缩放比例（相对于初始时的100%大小），从50%至400%。

传参：( scale )

名称类型/格式描述
scaleFloat相对于原来的100%大小，流程图显示缩放比例。取值0.5~4(50%至4倍大小)。

this.destrory() 返回值：void

设计器实例销毁自己。

传参：(无)

this.bindHeadBtnEvent() 返回值：void

对头部栏自定义按钮的事件绑定，可用来对另行加入的头部按钮自定义功能；但无法覆盖系统自带按钮类型的事件钩子定义。

传参：( funcJson )

名称类型/格式描述
funcJsonJsonMap对头部栏另行自定义类型按钮的事件绑定map集合,key为按钮类型名，value为方法实现定义

this.setTitle() 返回值：void

设置当前已载入的流程图的新名称。

传参：( text )

名称类型/格式描述
textString流程图的新名称。

this.setNodeCustoms() 返回值：void

在载入流程数据前设定特殊类型节点的CSS背景图及宽高属性。

传参：( json )

名称类型/格式描述
jsonJson Object对某些特殊类型节点的完全样式自定义配置，详见“数据结构定义”章节中的NodeCustoms。

this.initNavMap() 返回值：void

初始化导航图功能。(仅Js原生版使用)

要调用该方法，需扩展引入navmap.js文件。

传参：( selector, width, height )

名称类型/格式描述
selectorString要渲染为设计器导航图的Dom容器选择器，格式与jQuery的选择器格式相同。
widthNumber导航图的宽度，默认为null或未定义时，直接取导航图容器的原宽度。
heightNumber导航图的高度，默认为null或未定义时，直接取导航图容器的原高度。

this.refreshMap() 返回值：void

根据载入的流程图内容刷新导航图显示数据。

要调用该方法，需扩展引入navmap.js文件。

传参：(无)

数据操作接口

this.setName() 返回值：void

设置当前已载入的流程图中，某个节点/连线/分组泳道的显示名称。

传参：( id, name, type )

名称类型/格式描述
idString要操作的元素的id。
nameString要设置的新名称。
typeString要操作的元素的类型；有'node'节点、'line'连线、'area'分组泳道三种。

this.transNewId() 返回值：void

变更元素的id,一般用于快速保存后,将后台返回新元素的id更新到页面中。

传参：( oldId, newId, type )

名称类型/格式描述
oldIdString要操作的元素的id。
newIdString要设置的新id值。
typeString要操作的元素的类型；有'node'节点、'line'连线、'area'分组泳道三种。

this.markItem() 返回值：void

用醒目的颜色(已经自带)标注/取消标注一个节点或转换线，常用于显示重点或跟踪流程的进度。该功能在只读模式下时非常有实际应用意义。
如果要自定义标注时的颜色，请修改GooFlow.colors.mark。

传参：( id, type, mark )

名称类型/格式描述
idString要标注的元素id。
typeString要标注的元素种类，有节点'node'、连线'line'两种。
markBooleantrue表示标注；false表示取消标注。

this.addNode() 返回值：void

增加一个节点。

传参：( id, data )

名称类型/格式描述
idString新增节点的id。
dataNodeItem新增节点的详细数据信息。详见“数据结构定义”章节中的NodeItem类型。

this.moveNode() 返回值：void

移动一个节点到一个新的位置。

传参：( id, left, top )

名称类型/格式描述
idString要操作节点的id。
leftNumber节点新位置相对于工作区的左边距；单位像素。
topNumber节点新位置相对于工作区的上边距；单位像素。

this.resizeNode() 返回值：void

重新设置节点的尺寸；但是带有' round'类型的节点（即圆形节点）不支持该方法。

传参：( id, width, height )

名称类型/格式描述
idString要操作节点的id。
widthNumber节点的新宽度，单位像素；最小104px。
heightNumber节点新高度，单位像素；最小26px。

this.delNode() 返回值：void

删除一个节点。

传参：( id )

名称类型/格式描述
idString要删除节点的id。

this.addLine() 返回值：void

增加一条连线。

传参：( id, data )

名称类型/格式描述
idString新增连线的id。
dataLineItem新增连线的详细数据信息。详见“数据结构定义”章节中的LineItem类型。

this.setLineStyle() 返回值：void

重新设置连线的显示风格。

传参：( id, newStyle )

名称类型/格式描述
idString要操作连线的id。
newStyleString连线的新显示风格。取值有四种：'direct'有向实线；'dirDashed'有向虚线；'segment'无向实线；'segDashed'无向虚线。

this.setLineType() 返回值：void

重新设置连线的类型。

传参：( id, newType, M )

名称类型/格式描述
idString要操作连线的id。
newTypeString连线的新类型。取值有三种：'sl'直线；'lr'中段可左右移动的折线；'tb'中段可上下移动的折线有直线。
MNumber 一种抽象值，单位为像素。
当type='lr'时为必填项，表示中段线相对于工作区的X坐标值；
当type='tb'时为必填项，表示中段线相对于工作区的Y坐标值。
当type='sl'时无任何意义。

this.setLineM() 返回值：void

设置折线第二条中段的X坐标值（可左右移动时）或Y坐标值（可上下移动时）；直线不支持此方法。

传参：( id, M )

名称类型/格式描述
idString要操作连线的id。
MNumber新的M值。当type='lr'时表示中段线相对于工作区的X坐标值； 当type='tb'时表示中段线相对于工作区的Y坐标值。

this.setLineMl() 返回值：void

设置折线第一条中段的X坐标值（可左右移动时）或Y坐标值（可上下移动时）；直线不支持此方法。

传参：( id, M )

名称类型/格式描述
idString要操作连线的id。
MNumber新的M值。当type='tb'时表示中段线相对于工作区的X坐标值； 当type='lr'时表示中段线相对于工作区的Y坐标值。

this.setLineMr() 返回值：void

设置折线第三条中段的X坐标值（可左右移动时）或Y坐标值（可上下移动时）；直线不支持此方法。

传参：( id, M )

名称类型/格式描述
idString要操作连线的id。
MNumber新的M值。当type='tb'时表示中段线相对于工作区的X坐标值； 当type='lr'时表示中段线相对于工作区的Y坐标值。

this.moveLinePoints() 返回值：void

变更连线两个端点所连的节点。

传参：( id, newStart, newEnd )

名称类型/格式描述
idString要操作连线的id。
newStartString新的起始节点的id；如果为null则表示原端点不变。
newEndString新的目标节点的id；如果为null则表示原端点不变。

this.delLine() 返回值：void

删除一条连线。

传参：( id )

名称类型/格式描述
idString要删除连线的id。

this.setItemExtra() 返回值：void

快捷批量设置某个单元的自定义扩展属性，用户可用其设置只与具体业务相关的独有属性。

传参：( id, type, extra, [rerender] )

名称类型/格式描述
idString元素的id
typeString元素的类型，有'node'节点、'line'连线、'area'分组泳道三种。
extraJson Object用户需要扩展的只与具体业务相关的独有属性集合。
rerenderBoolean是否重新渲染绘图区上元素的显示，默认否；vue3版本下无此传参，因为均为自动渲染。

this.delItemExtra() 返回值：void

快捷批量删除某个单元的自定义扩展属性，用户可用其删除只与具体业务相关的独有属性。

传参：( id, type, propKeys, [rerender] )

名称类型/格式描述
idString元素的id
typeString元素的类型，有'node'节点、'line'连线、'area'分组泳道三种。
propKeysArray[String]用户需求清空掉的只与具体业务相关的独有属性集合。
rerenderBoolean是否重新渲染绘图区上元素的显示，默认否；vue3版本下无此传参，因为均为自动渲染。

this.setItemColor() 返回值：void

设置某个单元的自定义颜色。

传参：( id, type, color )

名称类型/格式描述
idString元素的id
typeString元素的类型，有'node'节点、'line'连线两种。
colorString元素的新背景颜色或线条颜色。

this.setItemFontColor() 返回值：void

设置某个单元的自字义文字颜色。

传参：( id, type, color )

名称类型/格式描述
idString元素的id
typeString元素的类型，有'node'节点、'line'连线两种。
colorString元素的新文字颜色。

批量元素操作接口

this.batchMarkItem() 返回值：void

用醒目的颜色(已经自带)批量地标注/取消标注多个节点/连线元素。需扩展引入multi.js文件。
如果要自定义标注时的颜色，请修改GooFlow.colors.mark。

传参：( nodeIds, lineIds, mark )

名称类型/格式描述
nodeIdsString[]所有要标注的节点的id组成的数组。
lineIdsString[]所有要标注的连线的id组成的数组。
markBoolean操作类型：true表示标注，false表示取消标注。

this.batchAddItem() 返回值：void

批量地增加多个节点/连线元素。需扩展引入multi.js文件。

传参：( nodes, lines )

名称类型/格式描述
nodesJsonMap要新增的多个节点的信息。
在该JsonMap中，key为节点的id，value为节点详细信息，属于NodeItem类型数据。
linesJsonMap要新增的多个连线的信息。
在该JsonMap中，key为连线的id，value为连线详细信息，属于LineItem类型数据。

this.batchMove() 返回值：void

批量地移动多个节点/连线元素。需扩展引入multi.js文件。

传参：( nodeIds, lineIds, moveX, moveY )

名称类型/格式描述
nodeIdsString[]所有要移动的节点的id组成的数组。
lineIdsString[]所有要移动的连线的id组成的数组。
moveXNumber所有元素相对于原位置的水平位移，单位像素。
moveYNumber所有元素相对于原位置的垂直位移，单位像素。

this.batchDelete() 返回值：void

批量地删除多个节点/连线元素。需扩展引入multi.js文件。

传参：( nodeIds, lineIds)

名称类型/格式描述
nodeIdsString[]所有要删除的节点的id组成的数组。
lineIdsString[]所有要删除的连线的id组成的数组。

数据I/O接口

this.getItemInfo() 返回值：JsonMap

根据id和类型，返回相应的流程图元素单元数据。

返回的Map，根据入参type类型的不同，有NodeItem节点、LineItem连线、AreaItem分组泳道、'memo'便笺四种

传参：( id, type )

名称类型/格式描述
idString元素的id
typeString选填项：元素的类型，有'node'节点、'line'连线、'area'分组泳道、'memo'便笺四种。
当本项不填写时，会遍历所有节点、连线、泳道元素，并返回找到的第一个id为传参id值的元素数据。

this.loadData() 返回值：void

载入一组JsonMap格式的流程图数据。

传参：( jsonData )

名称类型/格式描述
jsonDataFlowChart要载入的整张流程图数据。详见“数据结构定义”章节中的FlowChart类型。

this.exportData() 返回值：FlowChart

把画好的流程图结果导出到一个本函数返回的变量中（其实也可以直接访问GooFlow实例的$nodeData,$lineData,$areaData这三个JsonMap属性）； 关于返回数据的结构体，详见“数据结构定义”章节中的FlowChart类型。

传参：(无)

this.exportAlter() 返回值：FlowChart

只把本次编辑的流程图中作了变更(包括增删改)的元素导出到一个变量中,以方便用户每次编辑载入的流程图后只获取变更过的数据； 关于返回数据的结构体，除了没有title属性外，其余均详见“数据结构定义”章节中的FlowChart类型。

传参：(无)

this.clearData() 返回值：void

清空工作区及已载入的流程图数据。

传参：(无)

this.exportDiagram() 返回值：void

专门负责导出流程图文件并让用户下载的扩展包方法；注意：
1、当使用css sprite方式定义节点图标时，各节点图标只支持全都只来自同一个背景图片文件！
2、要调用该方法，需扩展引入 export.js 文件

传参：( filename )

名称类型/格式描述
filenameString用户下载导出图片时初始的文件名。

this.print() 返回值：void

打印流程图或将流程图另存为PDF的扩展包方法；注意：
1、不支持IE8及以更早的IE浏览器。
2、要调用该方法，需扩展引入 print.js 文件

传参：( scale )

名称类型/格式描述
scaleFloat流程图输出打印时的缩放比例，取值0.5~4(50%至4倍大小)；不填时默认为1。

this.exportBpmnXML() 返回值：String

把流程图中的数据以Bpmn2.0格式的XML字符串输出。注意：
要调用该方法，需扩展引入 xml.js 文件

传参：( nodeMaps )

名称类型/格式描述
nodeMapsJson Object定义了流程图中节点的各种定义类型需转换成的XML标签名。
key是节点在Json中的类型，value是节点在XML中的标签名。

this.exportDataFile() 返回值：void

把流程图中的数据以json格式文件或Bpmn2.0的XML格式文件导出并让用户下载。注意：

事务控制接口

以下均是在初始化的参数options.useOperStack=true时,才存在的方法接口。

this.pushOper() 返回值：void

仅供内部方法调用的方法:把对工作区内的数据单元编辑操作(增/删/改/重命名/移动/标注等)加入整条管理栈中,进行撤销/重做的控制；管理栈最多能放50条历史操作记录。

传参：( funcName, paras )

名称类型/格式描述
funcNameString被push入管理栈的操作方法名
parasArray[]被push入管理栈的操作方法的传参，从前至后依次放入一个数组中。

this.pushExternalOper() 返回值：void

外部的方法加入到GooFlow实例的事务操作堆栈中，在过后的undo/redo操作中可以进行控制，一般用于对流程图以外的附加信息进行编辑的事务撤销/重做控制；
注意：为了让外部方法能够被 undo/redo，需要在编写这些外部方法实现时，加入对该方法执行后效果回退的另一个执行方法的pushExternalOper。

传参：( func, jsonPara )

名称类型/格式描述
funcFunction被push入管理栈的外部执行方法对象。
jsonParaJson Object为外部方法仅有的一个面向字面的JSON传参或者数据,由JSON对象或数组带入所有要传的信息。

this.undo() 返回值：void

撤销最近一次操作。

传参：(无)

this.redo() 返回值：void

重做最近一次被撤销的操作。

传参：(无)

传参：( funcJson )

名称类型/格式描述
funcJsonJsonMap要扩展出的方法接口定义map集合,key为方法接口名，value为方法实现定义。
